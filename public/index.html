Client Status Server

Панель для безопасного списка клиентов: клиенты отправляют heartbeat, сервер хранит их статус и показывает online/offline без удалённых команд.

Авторизация

Для маршрутов /api/* нужен токен из переменной окружения TOKEN.

Server:
Public health: GET /api/health

Сводка

Краткое состояние сервера и общего списка клиентов.

0 Всего клиентов
0 Онлайн
0 Оффлайн
0 TTL, сек

Тестовый Heartbeat

Форма для ручной регистрации или обновления клиента.

API

Короткая схема маршрутов статуса клиентов.

GET /api/health
Публичный healthcheck сервера без токена.
POST /api/heartbeat
Body: { "clientId", "name", "version", "group", "status", "meta" }
Создаёт или обновляет клиента и время последнего heartbeat.
GET /api/clients
Возвращает весь список клиентов, online/offline и TTL.
GET /api/clients/:clientId
Возвращает одного клиента по `clientId`.
DELETE /api/clients/:clientId
Удаляет клиента из списка.
